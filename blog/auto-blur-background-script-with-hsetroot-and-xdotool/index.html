Auto-blur background script with hsetroot and xdotool
ad
Apr 8, 2018
2 minutes read

so yesterday when i was trying out this rather interesting wm (it has a flipping rainbow!!!!), i found this simple auto blur script that’s also caught my interest (directly copied) :

#!/bin/bash
win="$(lsw)"
if [[ $win ]]; then
	hsetroot -fill $HOME/wall.jpg -blur 15
else
	hsetroot -fill $HOME/wall.jpg -blur 15
fi

it’s simply uses lsw command that according to the wmutils readme is a utility to list window and then run hsetroot command to blur the image and set the output as background.

looking for alternatives

first i thought i would need to replace lsw with a more common alternatives, one i could think of is wmctrl but running wmctrl -l will list all opened windows on every workspaces, you could probably specify a current workspace for that but i doubt it would work on any wm without emwh support.

And the other one is using xdotool, at first i thought i would use its function to output window title but the problem is that any title contains character such as [ ] or / will get ignored, idk. luckily though, upon re-reading through the manpages i found this wonderfully convenient command to simply output the current active window id, which is exactly what i need as it’s only contains numbers :

xdotool getactivewindow

now rewrite the script

#!/bin/bash
#

# write the function first
  lalala() {
    win="$(xdotool getactivewindow)"
    
    # check if the command above output something and run hsetroot 
    if [[ $win -eq 0 ]]; then
        hsetroot -fill /path/yourimage.png
    else
        hsetroot -fill /path/yourimage.png -blur 4
    fi
}

# and then loop it
_main() {
    while true ; do
        echo "$(lalala)"
        sleep 1
    done
} > /dev/null 2>&1

# now call the main function
_main"$@"
